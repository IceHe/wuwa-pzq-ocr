# 后端 OCR 识别方案备忘

这份文档保留浏览器版之前的后端识别设计，便于后续如果重新需要服务端 OCR 时参考，不代表当前仓库仍然启用这套实现。

## 当前仓库边界

截至 2026-04-02，当前运行链路是：

- 前端在浏览器内执行 OCR
- 后端只负责页面分发、识别记录入库、历史记录查询、截图上传
- 数据落到 PostgreSQL `wuwa_rebuild_log`
- 原始截图落到仓库根目录 `images/`

也就是说，下面的内容是“如果未来要恢复服务端 OCR，可以按这个方向重建”的设计备忘，不是当前线上逻辑说明。

## 目标界面

- 游戏: 《鸣潮》
- 场景: 声骸重构结果界面
- 输入: 单张截图
- 输出:
  - `user_id`
  - 左侧 5 条原词条
  - 右侧 5 条新词条
  - 每条词条的名称、数值、档位、锁定状态

## 推荐技术栈

- 图像处理: `OpenCV`
- OCR: `PaddleOCR`
- 后处理纠错: Python 内置模糊匹配或 `RapidFuzz`
- Web API: `Flask` 或 `FastAPI`

## 推荐流水线

1. 读取截图并统一到参考坐标系
2. 用锚点或参考布局定位左右词条面板
3. 将两个面板各切成 5 行 ROI
4. 对每行切分名称区和值区
5. 锁定状态优先用模板匹配，缺模板时回退颜色启发式
6. 名称和值分别做 OCR
7. 用限定词条集合做名称纠错
8. 将数值吸附到合法副词条档位
9. 提取右下角 `特征码: {数字}` 作为 `user_id`

## 参考布局参数

以 `1920x1080` 样本图为基准:

- 参考宽高: `1920 x 1080`
- 中间箭头框: `x=928, y=628, w=66, h=62`
- 左侧面板: `x=345, y=523, w=518, h=271`
- 右侧面板: `x=1053, y=523, w=518, h=271`
- 行高: `47`
- 行间距: `9`

文本切分参数:

- 名称左边距: `36`
- 名称宽度: `275`
- 文本上下边距: `3`
- 左侧数值右偏移: `145`
- 左侧数值宽度: `110`
- 右侧数值右偏移: `140`
- 右侧数值宽度: `150`

特征码裁剪区域:

- 宽度: `420`
- 高度: `100`
- 从右下角开始裁

当截图分辨率变化时，可按:

`scale = min(width / 1920, height / 1080)`

然后对上述参数统一缩放。

## 锁定状态识别

优先方案:

- 准备 `lock.png`
- 对每行左侧小图标区域做 `matchTemplate`

回退方案:

- 取每行最左侧约 `1/7` 区域
- 转 HSV
- 统计高亮像素比例
- 当高亮比例超过阈值时判为锁定

## OCR 预处理

对名称区和值区分别处理:

- 放大到目标高度约 `96px`
- 灰度化
- 对比度增强
- 轻度模糊
- Otsu 二值化
- 外围补黑边

推荐先跑原图 crop，失败后再跑预处理版本。

## 词条集合

建议限制在以下副词条集合内做纠错:

- 暴击
- 暴击伤害
- 攻击
- 生命
- 防御
- 共鸣效率
- 普攻伤害加成
- 重击伤害加成
- 共鸣技能伤害加成
- 共鸣解放伤害加成
- 固定生命
- 固定防御
- 固定攻击

其中 `攻击 / 防御 / 生命` 需要根据数值是否带 `%` 判断是百分比词条还是固定词条。

## 数值吸附

服务端实现建议维护每个词条的合法档位表。

示例:

- 暴击: `6.3, 6.9, 7.5, 8.1, 8.7, 9.3, 9.9, 10.5`
- 暴击伤害: `12.6, 13.8, 15.0, 16.2, 17.4, 18.6, 19.8, 21.0`
- 固定攻击: `30, 40, 50, 60`

OCR 得到的数值如果与最近合法档位差距在容忍范围内，则吸附到该档位，并输出 `tier`。

## User ID 提取

从右下角裁剪区域 OCR 后:

- 优先匹配 `特征码[:：]?(\d{6,})`
- 若无标签命中，则取最长的连续数字串

## API 设计建议

若未来恢复后端识别，可保留如下接口:

- `POST /api/recognize`
  - 请求: `multipart/form-data`, 字段 `image`
  - 响应: JSON

示例响应:

```json
{
  "filename": "demo.png",
  "result": {
    "user_id": "100491393",
    "original_stats": [],
    "new_stats": []
  }
}
```
